## 规则

所有法律文书起草、修订的输出，必须遵循**按业务场景分条**原则。这是默认行为，不再需要提醒。

## 具体标准

**判断方法**：写完条款列表后，问"如果我用一段话跟同事讲清这个场景，这段话里会不会同时涉及责任、费用、流程？" 会 → 合并为一条。

**正例（一个场景=一条）**：
```
第三条　下架处理
3.1 责任归属（甲方原因导致下架）
3.2 处理方式（乙方配合重新审核，不保证结果）
3.3 对应费用（每次重提审10万元，甲方承担）
```

**反例（按法律概念拆分）**：
```
第四条　重新提审费用（只写钱）
...
第七条　下架责任（只写责任，要引到第四条才知道费用）
```

**适用范围**：合同、补充协议、律师函、法律意见书、投诉函等一切法律文书。

**Why:** 用户在审核补充协议时指出"重提审费"和"下架责任"是同一件事，拆成两条让读者来回翻找。按场景组织后，阅读者按顺序读下来就能完整理解一个交易环节。

**How to apply:** 下笔前列出业务场景清单（如：上架→运营→下架→切包），每个场景一条。在每个场景条款内按自然因果顺序组织：触发条件→权利义务→对应费用。
