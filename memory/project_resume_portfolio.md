## AI管培生项目：传奇游戏公司法务智能体

**时间**：2026年5月
**角色**：AI管培生
**公司**：传奇游戏公司（传奇IP方向）

---

## 一、项目背景

传奇游戏公司法务部日常处理合同管理、传奇IP侵权诉讼、员工法律咨询、广告合规审查、玩家客诉等业务，流程依赖人工操作，存在归档耗时长、重复咨询多、诉讼节点分散等痛点。目标是构建一套法务智能体系统，实现法务工作的自动化与自助化。

---

## 二、项目架构

### 2.1 双智能体架构

| 维度 | 智能体A（内部法务） | 智能体B（员工自助） |
|------|-------------------|-------------------|
| 使用者 | 法务团队 | 全公司员工 |
| 推理引擎 | Claude Code / Aily | Aily 内置模型 |
| 工作流 | 三阶段校验 | 单轮简化分析 |
| 能力 | 合同终审、多合同对比、文书起草、诉讼管理、工伤指引、FAQ | 合同预审、纠纷建议、FAQ |
| 免责声明 | 不需要 | 强制输出 |
| 限制 | 无限制 | 20次/人/日 |

### 2.2 全链路系统架构

```
手机飞书 -> 飞书服务器 -> WebSocket -> Python relay
  -> 文件消息队列 -> Claude Code 调度器
    -> 法务智能体分析 -> 飞书回复

数据层：多维表格（合同归档 / 案件节点）+ 法务知识库
```

## 四、核心交付物

### 4.1 飞书AI桥接系统
- 搭建飞书Bot → 本地AI工作站的双向通信链路
- 基于WebSocket长连接，**事件驱动**架构，零空闲token消耗，3-5秒响应
- 双Bot独立部署（bot-1 + bot-2），共享AI技能层
- 权限模型：白名单机制（私聊/群聊）+ 管理员识别

### 4.2 法务智能体系统（Claude Code原型）
- **合同审核引擎**：三观四步法 + 四级风险分级（致命/重要/一般/轻微）
- **引用规范体系**：法可依原则（每条建议标注法律依据）+ 参考例文原文引用 + 二次核验
- **合同条款组织规范**：按业务场景分条，不按法律概念拆分
- **文档解析流水线**：部署MinerU OCR引擎（1.1GB模型），支持PDF/DOCX/图片 → Markdown
- 完成多次真实合同审核（CPS推广协议、iOS过包协议、服务器代充值协议、BOSS直聘服务合同）

### 4.3 Aily平台迁移
- 编写9份Aily配置文件：IDENTITY.md / SOUL.md / SOUL_B.md / USER.md / MEMORY.md / TOOLS.md / AGENTS.md / SKILL_ADAPTATION.md / MIGRATION_GUIDE.md
- 识别并映射法律知识库：12个法律领域 + 25个案由分析 + 200+条司法解释 + 7个方法论文件
- 设计5阶段迁移路径，确定混合架构方案（Aily对话前端 + Claude Code推理引擎）

### 4.4 产品需求文档
- 编写`PRD_法务智能体_v0.1.md`（575行），覆盖产品概述、双智能体架构、功能需求（M1-M4 + A1-A7 + B1-B3）、非功能需求、权限模型、技术架构、数据模型、验收标准、实施路径（6阶段里程碑）

## 六、方法论沉淀

- **合同审核工作流**：合同逐条分析法（默认）| 三观四步法（按需）
- **法律分析框架**：IRAC（Issue-Rule-Application-Conclusion）
- **引用规范**：法可依原则 + 参考例文原文引用 + 二次核验（上网确认法条时效）
- **OCR处理**：MinerU + PaddleOCR，PDF/DOCX/扫描件 → Markdown
- **多智能体编排**：A/B共享知识库 + 严格权限隔离
- **项目推进框架**：看→拆→算→推（发现机会→拆解步骤→估算ROI→快速验证）
