# 法务智能体 — 项目概述

游戏行业法律 AI 助手的迁移包交付项目。采用"AI 优先"的开发流程：
Claude Code 原型验证 → 飞书Bot 内测 → 迁移包打包 → 目标平台集成。

## 项目架构

```
法务智能体
├── 法务助手迁移包 (智能体A)    ← 内部法务团队使用
│   ├── 合同终审                 三阶段工作流 + IRAC + 六级风险
│   ├── 多合同对比               差异分析 + 风险评级
│   ├── 法律文书起草             10+ 种文书类型
│   ├── 诉讼节点管理             全生命周期跟踪
│   ├── 法律知识 FAQ             模块化知识库
│   └── 工伤指引                 六环节端到端
│
└── 员工弱化包 (智能体B)         ← 全员自助使用
    ├── 合同预审                 快速红线扫描
    ├── 纠纷路径建议             通用流程指引
    └── 法律知识 FAQ             轻量问答
```

## 开发 SOP

采用五阶段流水线：
1. **Phase 0**: 知识资产化（知识库构建）
2. **Phase 1**: Claude Code 原型验证（功能快速验证）
3. **Phase 1.5**: 功能验收（达标准入）
4. **Phase 2**: 飞书Bot 测试（实际业务测试）
5. **Phase 3**: 功能固化 + 知识库清洗
6. **Phase 4**: 打包迁移 + 平台适配
7. **Phase 5**: 回归验证 + 文档

详见 `docs/法务智能体开发 SOP.md`。
