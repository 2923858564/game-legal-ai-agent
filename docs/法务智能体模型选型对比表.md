## 三、关键维度详细分析

### 3.1 上下文长度（Context Length）

这是法务场景最关键的技术指标，原因如下：

- 一份完整的推广合作协议通常 **30-80 页**，含扫描件附件更可达 **100+ 页**
- 多合同对比需同时加载 **2~3 份合同全文**
- 三阶段校验工作流（初步分析→法律校验→反思修正）会累积大量中间输出

| 场景 | 预估 Token 消耗 | 推荐模型 |
|------|---------------|---------|
| 单份合同终审（30页） | ~30K | 所有模型均可 |
| 单份合同终审（100页） | ~100K | 需 128K+ 模型 |
| 2份合同对比 | ~100K-200K | 需 128K-256K 模型 |
| 3份合同对比+历史对话 | ~200K-400K | 需 256K-1M 模型 |
| 诉讼分析+全文材料 | ~200K+ | 需 256K+ 模型 |

**结论**：deepseek-v4-flash（1M 上下文）在此维度有显著优势。

### 3.2 法律推理精度

从合同审核方法论体系的实践来看：

- **deepseek-v4-pro** + 完善的引用规范提示词，法律推理精度最高，法条引用准确率可达 90%+
- **deepseek-v4-flash** 在法律推理上略低于 pro 但差距不大，性价比极高
- **qwen3.6-max** 在国内法律场景有良好表现，尤其在中文法律条文理解上
- **doubao-1.5-pro** 在 OCR 和文档理解上有独特优势（已验证处理扫描件合同）
- **doubao-pro-128k** / **qwen3.6-plus** 中等偏上，适合 B 智能体（员工侧）的轻量场景

### 决策建议

1. **短期（当前阶段 → 2-3 个月内）**：维持 **deepseek-v4-flash** 当前方案。成本低、1M 上下文、法律推理精度高，已验证适配法务工作流。

2. **中期（3-6 个月）**：评估 **火山引擎混合方案** 作为国内合规替代，重点测试：
   - doubao-1.5-pro-256k 在长合同场景下的实际推理精度

   - OCR 能力与当前法务流程的集成度
