### 能力映射详细说明

#### 1. 智能路由 → Aily 实现

**原 Skill 方式**：通过 trigger words 列表在 prompt 中定义路由规则。
**Aily 方式**：直接将路由规则写入 SOUL.md 的"场景识别"部分。

```
用户输入
  → Aily模型读取SOUL.md中的路由规则
  → 根据触发词自动匹配合适的分析模式
  → 按匹配模式对应的流程执行
```

**注意事项**：Aily 没有类似 Claude Code 的独立路由引擎，路由依赖模型自身的识别能力。建议：
- 在 SOUL.md 中将触发词列表写详细
- 在知识库中按场景分组，模型会根据上下文自动找到最相关的知识
- 如果路由不准，可以补充更多业务场景范例

#### 2. 三阶段工作流 → Aily 实现

**原 Skill**：Claude Code 依次执行 Phase 1 → Phase 2 → Phase 3，在同一个对话中自我校验。

**Aily 方式**：
- **智能体A**：将三阶段工作流写入 SOUL.md，让模型在回答时自动遵循
- **智能体B**：不需要三阶段，采用简化的单轮分析流程

```
Aily 中的三阶段执行：
Phase 1 初步分析 → 模型按 IRAC 框架直接输出
Phase 2 法律校验 → 模型在输出前自动检查引用的法条（依赖知识库中的法规文件）
Phase 3 反思修正 → 模型在最终输出前自我修正
```

**模型差异**：Aily 内置模型的自我反思能力可能不如 Claude Code。建议：
- 智能体A：先用混合架构（Aily 前端 + Claude Code 推理），验证 Aily 模型效果后再决定是否迁移
- 智能体B：Aily 纯云端的简化分析已经够用

#### 3. 合同审核方法论 → Aily 实现

完整的三观四步法、四级风险分级、按业务场景分条原则——全部写入 SOUL.md。

对于多合同对比功能，需要在 prompt 中明确定义对比模板：

```
多合同对比输出模板：

| 对比维度 | 合同A | 合同B | 合同C | 差异分析 |
|---------|-------|-------|-------|---------|
| 合同金额 | | | | |
| 合作期限 | | | | |
| 违约责任 | | | | |
| 管辖约定 | | | | |
| 知识产权归属 | | | | |
| 风险等级 | | | | |
| 异常/冲突点 | | | | |

附加输出：冲突清单、建议合并方案、风险变化说明
```

对于合同条款组织规范，也直接写入 SOUL.md：

```
合同起草/修订输出时，按业务场景分条，不按法律概念拆分。
正例：第三条 下架处理 → 3.1责任归属 → 3.2处理方式 → 3.3对应费用
反例：第四条 费用（只写钱）... 第七条 责任（读者来回翻）
```

## Aily 限制与应对

### 已知限制

1. **Aily 内置模型法律推理精度**：可能不如 Claude Code
   - **应对**：混合架构（Claude Code 推理 + Aily 前端），或选择 Aily 支持的高精度模型

2. **Aily 单轮上下文窗口**：可能小于 Claude Code
   - **应对**：控制对话长度，知识库文件拆分到合理大小（建议每个文件不超过2000字）

3. **Aily 知识库检索精度**：依赖文件组织和命名
   - **应对**：按照 MEMORY.md 的目录结构精细组织，文件命名包含业务关键词

4. **Aily 不支持自定义代码执行**
   - **应对**：原 Skill 中的代码逻辑（如文件解析）移到外部工具或混合架构中
