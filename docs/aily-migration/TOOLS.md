
# 工具与能力配置

## 概述

以下工具需要在 Aily 平台中配置，赋予智能体与飞书生态交互的能力。

## 工具清单

### 1. 文档读取与导出

这是实现多合同对比和合同审核的关键工具。

| 功能 | Aily 实现方式 | 用途 |
|------|-------------|------|
| 读取消息中的文件 | Aily 对话文件上传（原生支持） | 用户直接上传合同文件 |
| 云文档导出为文本 | 飞书云文档导出API | bot中已有文档的提取 |
| docx全文提取 | Aily AI 原生文档理解 | 合同条款识别和对比 |

**工作流程**：
```
用户上传合同文件（docx/pdf）
  → Aily 内置文件处理器自动提取文本
  → 智能体读取文本内容进行分析
  → 输出审核报告/对比表
```

**关于扫描件 PDF**：
- Aily 内置模型对扫描件 PDF 的文字提取能力有限
- 建议扫描件通过 Claude Code 视觉能力处理（混合架构方案）
- 如果使用纯 Aily 方案，扫描件需要配合飞书文档 OCR 能力

### 2. 多维表格（Bitable）读写

用于合同归档和诉讼节点管理。

| 操作 | API / Aily 能力 | 用途 |
|------|----------------|------|
| 读取记录 | Aily Bitable 插件 | 查询归档合同、案件信息 |
| 写入记录 | Aily Bitable 插件 | 审批完成后自动归档 |
| 更新记录 | Aily Bitable 插件 | 诉讼节点更新 |
| 查询记录 | Aily Bitable 插件 | 按条件检索合同/案件 |

**Aily 配置方式**：在 Aily 插件市场中启用"多维表格"插件，授权对应的表格权限。

### 3. 审批连接器

用于合同审批完成后的自动归档。

| 触发 | 动作 | 用途 |
|------|------|------|
| 审批实例完成事件 | 读取审批表单字段 | 获取合同信息 |
| 自动写入 | 写入多维表格 | 归档合同数据 |

**Aily 配置方式**：飞书管理后台 → 审批 → 连接器 → 新建自动化流程。触发条件选"审批完成"，执行动作选"写入多维表格"。

### 4. 飞书搜索（可选）

用于检索内部法律文档、过往合同、公司规范。

| 功能 | 用途 |
|------|------|
| 搜索云文档 | 查找公司知识库中的法律文档 |
| 搜索多维表格 | 快速定位历史合同和案件 |

### 5. 消息通知（可选）

用于诉讼节点提醒和复杂场景转法务。

| 功能 | 用途 |
|------|------|
| 发送消息给用户 | 节点到期提醒 |
| @指定法务人员 | B智能体复杂场景转人工 |

## 工具权限控制

| 工具 | 智能体A | 智能体B |
|------|---------|---------|
| 文档读取 | ✅ 完全可用 | ✅ 仅限用户上传的文件 |
| 多维表格（合同归档） | ✅ 读写 | ❌ 只读本人提交 |
| 多维表格（案件节点） | ✅ 读写 | ❌ 不可用 |
| 飞书搜索 | ✅ 可用 | ❌ 不可用 |
| 审批连接器 | ❌ 自动运行 | ❌ 自动运行 |
| 消息通知 | ✅ 可用 | ❌ 不可用 |

## 扫描件处理方案（多合同审核关键）

经测试，当前 Aily 内置模型对扫描件 PDF 的文本提取存在局限。建议方案：

**方案一：混合架构（推荐）** 
- Aily 收到扫描件后，通过工具调用将文件转发给 Claude Code
- Claude 利用视觉能力提取文字和版式信息
- 返回提取结果给 Aily 智能体进行法律分析

**方案二：预处理转换**
- 用户在飞书云文档中上传PDF → 使用"图片转文字"在线能力转成文本
- 将文本粘贴到对话框中触发分析

**方案三：第三方OCR接入**
- Aily 插件接入OCR服务（如百度OCR、阿里云OCR）
- 扫描件自动触发OCR识别 → 文本进入分析流程

建议先采用方案一（混合架构）过渡，后续评估方案三的可行性和成本。
