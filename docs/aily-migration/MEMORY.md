
# 知识库结构

## 概述

知识库是智能体的核心记忆。以下组织结构用于在 Aily 知识库中建立模块化的法律知识体系。

## 知识库管理原则

1. **模块化**：每个文件聚焦一个主题，避免大而全的"百科全书型"文件
2. **定期更新**：法条变更后及时更新对应文件，标注版本日期
3. **分区隔离**：游戏行业知识与通用法律知识分开存储，避免交叉污染
4. **参考例文专区**：所有参考例文必须是真实历史案例/话术原文，不得编造
5. **版本标注**：每个法律文件顶部标注最后更新日期和所依据的法律版本

## 在Aily中配置知识库的注意事项

1. Aily知识库支持上传Markdown文件，建议按上述目录结构逐个上传
2. 每个知识库分组的上下文窗口有限，控制单个文件在合理篇幅内（建议不超过2000字/文件）
3. 合同模板类可直接上传docx版实际合同范本
4. 案例库中的判决书原文，建议脱敏后上传
5. 知识库权限与智能体权限联动——智能体B不应访问法务内部案件数据和终审策略
