
# 飞书 Aily 迁移实施总指南

> 目标：将当前 Claude Code + cc-connect + 飞书Bot 架构中的全部法务能力迁移到飞书 Aily 平台

## 架构选择

### 方案对比

| 维度 | 方案一：纯Aily | 方案二：混合架构（推荐） | 方案三：全量Claude + Aily |
|------|--------------|----------------------|------------------------|
| 架构 | Aily 独立运行 | Aily对话前端 + Claude Code推理 | Aily做入口 + 全部由Claude处理 |
| 智能体A | Aily内置模型 | Claude Code推理 + Aily前端 | Claude Code |
| 智能体B | Aily内置模型 | Aily内置模型 | Claude Code |
| 运维 | 低（云端） | 中（需维护本地） | 高（需维护本地+网络） |
| 法律精度 | 待验证 | 高（Claude保障） | 最高 |
| 成本 | Aily费用 | Aily费用 + 本地电费 | Aily费用 + 较高API费用 |
| 适用阶段 | 正式上线（精度达标后） | 内测期·过渡方案 | 精度要求极高的场景 |

**推荐方案**：第二阶段采用**混合架构**，待 Aily 模型法律精度验证通过后再考虑迁移到纯 Aily。

## 迁移步骤

### 第一阶段：Aily 智能体B 上线（纯云端，先跑起来）

这是最容易的部分，也是员工侧最急需的能力。

**步骤**：
1. 在 Aily 后台创建"员工法律助手"智能体
2. 上传配置：
   - IDENTITY.md → 使用智能体B版本
   - SOUL_B.md → 简化版（不含三阶段工作流）
   - USER.md → 聚焦员工角色
   - 知识库：法律法规 + 游戏行业专项 + 工伤指引 + FAQ
3. 开启基础工具（文件上传、知识库）
4. 配置用量限制（20次/人/日）
5. 发布给全员测试

**预计耗时**：2-3天

### 第二阶段：Aily 智能体A 上线（混合架构，保障精度）

智能体A 是法务核心能力，精度要求高，采用混合架构。

**步骤**：
1. 在 Aily 后台创建"内部法务专家"智能体
2. 上传配置：
   - IDENTITY.md → 使用智能体A版本
   - SOUL.md → 完整版（含三阶段工作流、IRAC、合同审核全流程）
   - USER.md → 聚焦法务用户角色
   - 知识库：全部开放
3. 配置混合推理：
   - Aily 对话界面 → 用户提问 → 通过工具调用转发到 Claude Code
   - Claude Code 执行法律分析 → 返回结果给 Aily → 展示给用户
   - 或者：用户在 Aily 对话框中直接与 Claude 对话（Aily 对接 Claude API）
4. 配置文档处理工具：
   - 电子版文件：Aily 原生文件处理
   - 扫描件：转发到 Claude 视觉处理
5. 开启多维表格工具：
   - 合同归档表（只读/写入）
   - 案件节点表（读写）

**预计耗时**：1周

### 第三阶段：合同归档自动化（审批连接器）

与智能体独立运行，不影响对话能力。

**步骤**：
1. 飞书管理后台 → 审批 → 连接器
2. 新建自动化流程：
   - 触发条件：审批实例完成
   - 筛选条件：合同类审批（按审批模板ID）
   - 执行动作：写入多维表格的合同归档表
3. 在 Aily 中配置对应的 Bitable 插件，让智能体可查询归档数据

**预计耗时**：2天（含字段映射测试）

### 第四阶段：智能体A 纯Aily化（可选）

如果 Aily 内置模型法律精度验证达标，可以将智能体A也从混合架构迁移到纯Aily。

**验证条件**：
- 合同终审风险识别准确率 ≥ 85%（以法务复核为准）
- 引用规范符合率 100%
- 三阶段工作流可稳定执行

### 第五阶段：交付

1. 编写 SOP 和操作手册（法务+员工各一份）
2. 知识库更新与维护规范
3. 用量监控和成本评估
4. 正式切换：Bot下线通知，引导至Aily

## Aily 后台配置要点

### 创建智能体

飞书后台 → 智能伙伴(Aily) → 创建智能体

### 配置维度对应关系

| Aily 配置项 | 对应文件 | 用途 |
|------------|---------|------|
| 角色身份 | IDENTITY.md | 基础设定 |
| 性格与行为准则 | SOUL.md | 行为规则、方法论 |
| 用户信息 | USER.md | 用户画像 |
| 知识库 | MEMORY.md 目录结构 | 法律知识存储 |
| 能力配置 | SKILL_ADAPTATION.md | 技能映射 |
| 工具/插件 | TOOLS.md | 外部能力 |
| 智能体编排 | AGENTS.md | 多智能体 |

### 知识库上传方式

Aily 知识库支持：
- 直接上传 Markdown 文件（推荐）
- 关联飞书云文档
- 关联多维表格
- 自定义文本输入

**建议**：先用 Markdown 文件上传，后续过渡到云文档关联（便于法务人员直接更新）。

### 权限配置

1. 智能体A：仅允许法务团队可见可用
   - 飞书后台 → 智能体A → 权限设置 → 指定可见范围（法务部门）
2. 智能体B：全员可见可用
   - 飞书后台 → 智能体B → 权限设置 → 全员可用
3. 知识库权限联动：
   - 案件数据和诉讼信息仅智能体A可访问
   - 法律法规和FAQ两智能体共享

## 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| Aily 内置模型法律精度不足 | 引用规范无法保证 | 混合架构兜底，A智能体始终走Claude推理 |
| 扫描件文字提取困难 | 多合同对比受阻 | 混合架构中 Claude 视觉处理 + OCR 插件备选 |
| 审批连接器实际可用性 | 合同归档延迟 | 准备定时 API 拉取作为备选方案 |
| Aily 用量限制 | 高峰时段不可用 | 混合架构下 Bot 作为备用通道 |
| 知识库维护不到位 | 引用的法律可能过时 | 设置定期审核流程（季度更新） |
| 法务参与度不足 | 知识库质量不达标 | 跟岗产出标准化 SOP，降低维护门槛 |
