## 概述

**法务智能体**是一个为游戏行业法务场景设计的 AI Agent 迁移包，包含完整的法律分析知识库、工作流定义和行为规范。它最初为**传奇游戏公司**的法务部开发，采用"AI 优先"的迁移包交付模式。

### 核心能力

| 能力 | 说明 |
|------|------|
| **合同审查** | 逐条审核合同，四级风险分级（🔴致命/🟠重要/🟡一般/🔵轻微），附法律依据原文 |
| **多合同对比** | 2~N 份合同对比，输出差异与冲突清单 |
| **法律文书起草** | 起诉状、答辩状、律师函、仲裁申请等 10+ 种文书 |
| **案件分析** | 三阶段校验工作流（初步分析→法律校验→反思修正）+ IRAC 分析框架 |
| **诉讼节点管理** | 立案→举证→开庭→判决→执行全生命周期跟踪 |
| **工伤指引** | 认定→申报→赔偿→鉴定全流程指引 |
| **法律知识 FAQ** | 劳动法、合同法、广告法、知识产权等领域 |
| **纠纷路径建议** | 通用纠纷处理流程指引 |

### 双包架构

| 维度 | 法务助手迁移包（智能体A） | 员工弱化包（智能体B） |
|------|-------------------------|---------------------|
| 适用对象 | 法务团队 | 全员 |
| 核心能力 | 合同终审、文书起草、诉讼管理、工伤指引 | 合同预审、纠纷建议、FAQ |
| 推理引擎 | Claude / DeepSeek | Aily 内置模型 |
| 工作流 | 三阶段校验 | 单轮简化分析 |
| 使用限制 | 无限制 | 20次/人/日 |
| 免责声明 | 不需要 | 必须输出 |

## 快速开始

### 方式一：Claude Code 直接使用

```bash
# 克隆仓库
git clone https://github.com/2923858564/game-legal-ai-agent.git
cd game-legal-ai-agent

# 在 Claude Code 中使用
claude
```

AI 会自动读取 `CLAUDE.md` 中的配置，加载法律分析技能和工作流定义。

### 方式二：飞书智能体集成

将 `skills/china-lawyer-analyst/` 目录下的知识库和 `CLAUDE.md` 中的行为规范导入飞书 Aily 平台，配置对应的推理模型即可。详见 `docs/aily-migration/` 中的迁移指南。

## 核心技术规范

### 三阶段校验工作流

```
Phase 1: 初步分析
  案件类型识别 → 六段式要件清单 → IRAC 分析

Phase 2: 法律校验
  提取法律引用 → 检查法条有效性 → 判断新旧法适用

Phase 3: 反思修正
  标记需更新法规 → 更新分析 → 重新 IRAC → 修正后意见
```

### 引用规范

所有法律分析输出必须遵循：
- **法可依原则**：每条建议标注法律依据（名称 + 条文号 + 原文）
- **参考例文原文引用**：建议/方案/话术必须附例文原文
- **二次核验**：法条引用上网确认现行有效

### 合同四级风险分级

| 等级 | 含义 | 行动 |
|------|------|------|
| 🔴 致命 | 合同无效/重大违约/刑事责任 | 必须修改 |
| 🟠 重要 | 权利义务显著失衡 | 建议修改 |
| 🟡 一般 | 表述不清/操作不便 | 可以修改 |
| 🔵 轻微 | 非核心问题 | 仅供参考 |
