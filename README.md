## 概述

**法务智能体**是一个为游戏行业法务场景设计的 AI Agent 迁移包，包含完整的法律分析知识库、工作流定义和行为规范。它最初为**传奇游戏公司**的法务部开发，采用"AI 优先"的迁移包交付模式。

### 核心能力

| 能力 | 说明 |
|------|------|
| **合同审查** | 逐条审核合同，四级风险分级（🔴致命/🟠重要/🟡一般/🔵轻微），附法律依据原文 |
| **多合同对比** | 2~N 份合同对比，输出差异与冲突清单 |
| **法律文书起草** | 起诉状、答辩状、律师函、仲裁申请等 10+ 种文书 |
| **案件分析** | 三阶段校验工作流（初步分析→法律校验→反思修正）+ IRAC 分析框架 |
| **诉讼节点管理** | 立案→举证→开庭→判决→执行全生命周期跟踪 |
| **工伤指引** | 认定→申报→赔偿→鉴定全流程指引 |
| **法律知识 FAQ** | 劳动法、合同法、广告法、知识产权等领域 |
| **纠纷路径建议** | 通用纠纷处理流程指引 |

### 双包架构

## 快速开始

### 方式一：Claude Code 直接使用

```bash
# 克隆仓库
git clone https://github.com/2923858564/game-legal-ai-agent.git
cd game-legal-ai-agent

# 在 Claude Code 中使用
claude
```

AI 会自动读取 `CLAUDE.md` 中的配置，加载法律分析技能和工作流定义。

### 方式二：飞书智能体集成

将 `skills/china-lawyer-analyst/` 目录下的知识库和 `CLAUDE.md` 中的行为规范导入飞书 Aily 平台，配置对应的推理模型即可。详见 `docs/aily-migration/` 中的迁移指南。

## 核心技术规范

### 三阶段校验工作流

```
Phase 1: 初步分析
  案件类型识别 → 六段式要件清单 → IRAC 分析

Phase 2: 法律校验
  提取法律引用 → 检查法条有效性 → 判断新旧法适用

Phase 3: 反思修正
  标记需更新法规 → 更新分析 → 重新 IRAC → 修正后意见
```

### 引用规范

所有法律分析输出必须遵循：
- **法可依原则**：每条建议标注法律依据（名称 + 条文号 + 原文）
- **参考例文原文引用**：建议/方案/话术必须附例文原文
- **二次核验**：法条引用上网确认现行有效

### 合同四级风险分级

| 等级 | 含义 | 行动 |
|------|------|------|
| 🔴 致命 | 合同无效/重大违约/刑事责任 | 必须修改 |
| 🟠 重要 | 权利义务显著失衡 | 建议修改 |
| 🟡 一般 | 表述不清/操作不便 | 可以修改 |
| 🔵 轻微 | 非核心问题 | 仅供参考 |

## 审核输出示例（真实案例）

以下为 **CPS推广合作框架协议** 深度审核的实际输出摘要，展示三阶段工作流的完整输出格式。

### Phase 1：初步分析

对合同逐条审查后，识别出 8 个法律风险点，其中 **3 项致命风险**：

| 编号 | 问题 | 风险等级 |
|------|------|---------|
| Q1 | 用户退款/坏账全部由甲方承担，违反公平原则 | 🔴 |
| Q2 | 违约期间全部充值流水作为违约金，属"过分高于损失" | 🔴 |
| Q3 | 甲方合作方被投诉即构成违约，当月分成全部没收 | 🔴 |
| Q4 | 服务器成本由甲方承担且无上限 | 🟠 |

每个问题独立进行 **IRAC 分析**（Issue → Rule → Application → Conclusion），逐条展示原文、法律依据、要件分析和风险结论。

### Phase 2：法律校验

对所有引用的法律进行现行有效性确认：

| 法律/司法解释 | 校验结果 |
|--------------|---------|
| 《民法典》第6条（公平原则） | ✅ 现行有效 |
| 《民法典》第497条（格式条款无效） | ✅ 现行有效 |
| 《民法典》第585条（违约金调整） | ✅ 现行有效 |
| 法释〔2023〕12号第65条（违约金30%标准） | ✅ 最新司法解释 |

### Phase 3：反思修正

输出对比初版与当前版的差异，跟踪分析质量提升：

| 初版问题 | 修正措施 |
|----------|----------|
| 法条引用未附完整原文 | 全部引用附完整条文原文 |
| 未援引最新司法解释 | 补充法释〔2023〕12号第65条（2023年12月施行） |
| 未说明格式条款分析路径 | 补充《民法典》第496-497条分析框架 |

### 最终审核意见书

输出包含四部分：
- **🔴 需立即修改的条款**（第5.8条退款承担、第5.13条违约金、第5.14条连坐条款）
- **🟠 建议谈判修改的条款**（服务器成本、设备ID回流、间接损失等）
- **🟡 关注但不一定修改的条款**
- **💡 谈判策略建议**（含地位分析、优先级建议、谈判话术示例）

> 完整示例见 [`docs/规范输出参考模板.md`](docs/规范输出参考模板.md)
