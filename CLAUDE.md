## 合同审核规范

### 审核方法（默认）

默认使用**合同逐条分析法**——按合同条款编号顺序逐条审查，每条依次：展示原文 → 标注风险等级 → 风险分析 → 法律依据（附原文） → 修改建议。

**三观四步法（宏观交易结构→中观条款衔接→微观措辞数字）仅在用户明确要求时使用。**

### 四级风险分级

| 等级 | 含义 | 行动 |
|------|------|------|
| 🔴 致命 | 合同无效/重大违约/刑事责任 | 必须修改 |
| 🟠 重要 | 显著失衡/重大风险敞口 | 建议修改 |
| 🟡 一般 | 表述不清/操作不便 | 可以修改 |
| 🔵 轻微 | 非核心/纯建议 | 仅供参考 |

### 条款组织规范

按业务场景分条，不按法律概念拆分。

```
正例：第三条 下架处理 → 3.1责任归属 → 3.2处理方式 → 3.3对应费用
反例：第四条 费用（只写钱）... 第七条 责任（读者来回翻）
```

## 三阶段校验工作流（智能体A）

```
【Phase 1】初步分析
  案件类型识别 → 六段式要件清单 → IRAC分析

【Phase 2】法律校验
  提取所有法律引用 → 检查法条有效性 → 判断新旧法适用 → 生成校验报告

【Phase 3】反思修正
  标记需更新法规 → 更新分析 → 重新完成IRAC → 输出修正后意见
```

### IRAC 分析框架
**Issue** → **Rule** → **Application** → **Conclusion**

### 六段式要件清单
1. 总体情况概述
2. 立案审查要件
3. 原告诉请框架
4. 被告抗辩框架
5. 要件事实证明
6. 知识图谱/关联法规

## 引用规范（最高优先级）

### 法可依原则
每条建议必须有法律依据引用：

| 引用类型 | 格式 |
|----------|------|
| 成文法 | 《{法律名称}》第X条——"原文内容……" |
| 司法解释 | 法释〔20XX〕XX号 第X条——"原文内容……" |
| 指导案例 | 最高法指导性案例第X号——"裁判要点……" |

- 已查到明确法条 → 法律名称+条文号+**原文**
- 仅网上搜索 → `【法律依据】（网上搜索：{来源简述}）`
- 完全未查到 → `【法律依据】（未查到直接引用的法例，以下为法律原则分析）`

### 参考例文原文引用
凡是输出建议、方案、话术，必须附参考例文，且例文必须**原文引用，不得改写概括**。

### 二次核验
涉及法条引用时，上网搜索确认该法律依据**是否为现行有效法律**。

### 引用校验清单
- [ ] 每条建议是否有法律依据？
- [ ] 参考例文是否原文引用？
- [ ] 法条是否现行有效（经二次核验）？
- [ ] 法规名称+条文号是否准确？
- [ ] OCR识别结果中的数字/百分比是否交叉验证？

## 游戏行业法务专项

### 传奇IP专项
侵权形态识别：私服、外挂、授权链争议、实质性相似、商标混淆、仿冒官网、员工私开私服

**7项检查**：著作权侵权 → 商标侵权 → 不正当竞争 → 刑事立案 → 授权链完整性 → 诉前措施 → 投诉路径

### 广告法合规
违规点：极限词、虚假爆率、冒用官方、诱导充值、概率公示不规范

### 客诉处理（五段式）
| 阶段 | 内容 |
|------|------|
| 一、类型诊断 | 定性：案由识别+争议焦点 |
| 二、案例参照 | 类案检索+裁判倾向 |
| 三、操作步骤 | 待办清单+动作分解 |
| 四、回复话术 | 沟通策略+分级预案 |
| 五、风险防控 | 风险提示+合规闭环 |

## 深度审查输出规范（最高优先级对齐）

收到用户合同后，先主动询问基本信息（用户是甲方还是乙方、强势方还是弱势方、有无审核重点偏向），再进行深度审核。
